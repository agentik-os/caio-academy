# Phantom Registry -- Le Pool Anonymise de Profils CAIO

> Le "phantom" n'est plus Gareth. C'est le pool anonymise des profils CAIO du Registry. Gareth est la face publique (LinkedIn primary). Le Registry est le moat.

---

## Le Pivot Concept

**Ancienne vision (archivee dans `99-archive/07-phantom-empire-legacy/`) :** Gareth opere sous pseudonyme, separation complete identite / business.

**Nouvelle vision :** Gareth est **100% public** sur LinkedIn + CAIO Academy. Le "phantom" existe toujours, mais il designe maintenant les **membres du Registry** -- leurs profils sont anonymises et poolles pour former la matiere premiere du business moat.

---

## Architecture du Phantom Registry

```
+-- GARETH (public face) ----------------------+
|  LinkedIn + YouTube + CAIO Academy           |
|  Voix + vision + formation                    |
+-----------------------------------------------+
              |
              | poole, anonymise, package
              v
+-- REGISTRY (moat) ----------------------------+
|  Profils CAIO anonymises                      |
|  - Industrie, stack, revenu, geographie       |
|  - PAS de nom, PAS de photo                   |
|  Accessible: cohorts Academy + clients B2B    |
+-----------------------------------------------+
              |
              | alimente
              v
+-- PRODUITS ----------------------------------+
|  - Dashboards benchmarks CAIO                |
|  - Reports industrie (anonymes)              |
|  - Intros ciblees (peer-matching)            |
|  - Research-as-a-service                     |
+-----------------------------------------------+
```

---

## Pourquoi ce design

| Principe | Implementation |
|----------|----------------|
| Face = trust | Gareth visible = credibilite marque |
| Pool = moat | Plus il y a de profils, plus le Registry devient difficile a copier |
| Anonymat membre | Les CAIO participent sans exposer leur entreprise |
| Network effect | Chaque nouveau CAIO enrichit les benchmarks pour tous |
| Data rights | Les membres consentent au pooling anonymise a l'inscription |

## Roadmap Registry

### Phase 1 -- Seed (Q2 2026)
- 50 profils CAIO invites manuellement
- Formulaire d'inscription + consentement pooling
- Dashboard interne Gareth only

### Phase 2 -- Early Access (Q3 2026)
- Ouverture aux graduates de CAIO Academy
- Premier benchmark report publie (anonyme, preview)
- Waitlist publique ouverte

### Phase 3 -- Moat (Q4 2026+)
- 500+ profils
- API B2B pour clients enterprise
- Reports mensuels payants
- Peer-matching automatise

## Regles operationnelles

1. **Gareth ne se cache pas.** LinkedIn, YouTube, keynotes -- tout public.
2. **Les profils CAIO sont anonymises par defaut.** Nom de l'entreprise only si opt-in explicite.
3. **Pas de vente de donnees brutes.** Uniquement benchmarks aggregates + intros consenties.
4. **Le Registry n'est pas Gareth.** Si Gareth disparait demain, le Registry doit continuer via la communaute + contrats.
