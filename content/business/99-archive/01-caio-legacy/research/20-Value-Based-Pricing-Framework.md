# Value-Based Pricing Framework

*Comment calculer tes prix quand tu vends un systeme, pas du temps.*

## Le Probleme Fondamental

### Si tu vends du temps

| Element | Detail |
|---------|--------|
| Task | Build un dashboard |
| Temps reel avec Agentik OS | 4 heures |
| TJM freelance classique | 600 EUR/jour |
| Tu factures | 300 EUR (0.5 jour) |
| **Probleme** | Le client paie 300 EUR pour un dashboard qui vaut 5000-15000 EUR sur le marche. Tu te fais arnaquer par ta propre vitesse. |

### Si tu vends la valeur

| Element | Detail |
|---------|--------|
| Task | Build un dashboard |
| Valeur marche | 8000-15000 EUR |
| Tu factures | 5000-8000 EUR |
| Le client economise | 3000-7000 EUR vs marche |
| Ta marge | environ 95% (cout reel: environ 200 EUR d'outils) |
| **Resultat** | WIN-WIN: Client paie moins que le marche. Toi tu gagnes 15x plus qu'au temps passe. |

**Regle d'or : JAMAIS vendre du temps. TOUJOURS vendre le resultat.**

## Le Framework de Pricing en 4 Etapes

### Etape 1 : Calculer la Valeur Marche (ce que ca coute SANS toi)

Pour chaque service, pose la question : "Combien le client paierait-il avec les alternatives traditionnelles?"

#### Grille de Valeur Marche par Service

| Service | Alternative traditionnelle | Cout traditionnel | Delai traditionnel |
|---------|---------------------------|--------------------|--------------------|
| MVP SaaS complet | Equipe 3-5 devs | 80-200K EUR | 4-8 mois |
| Landing page pro | Agence web | 5-15K EUR | 3-6 semaines |
| Dashboard interne | Dev freelance | 15-40K EUR | 2-4 mois |
| App mobile | Equipe mobile | 50-150K EUR | 4-8 mois |
| Marketing setup complet | Agence marketing | 3-8K/mois | Ongoing |
| Sales automation | Sales consultant + outils | 5-15K setup + 2K/mois | 2-3 mois |
| Branding + Design system | Agence branding | 10-30K EUR | 4-8 semaines |
| Support chatbot + KB | Dev + content team | 10-20K EUR | 2-3 mois |
| Strategy + Business plan | Consultant McKinsey | 2-5K/jour | 2-4 semaines |
| Full operations setup | COO + Operations team | 80-150K/an | 3-6 mois |

### Etape 2 : Appliquer le "Discount Factor" (ta reduction vs marche)

Tu ne factures PAS le prix du marche. Tu factures **30-60% du prix marche**. C'est ce qui rend ton offre irresistible.

**Formule : Ton Prix = Valeur Marche x Discount Factor**

| Contexte client | Discount Factor |
|-----------------|-----------------|
| Client bien funde (startup avec levee) | 50-60% du marche |
| PME avec budget serre | 30-40% du marche |
| Premier client / case study | 20-30% du marche |
| Client premium (grand compte) | 60-70% du marche |

**Exemple concret :**

| Element | Detail |
|---------|--------|
| Service | MVP SaaS complet |
| Valeur marche | 120K EUR (equipe de 4, 5 mois) |
| Discount factor | 50% |
| Calcul brut | 60K? Non, trop eleve pour le percu. |
| Ajustement | Le client ne compare pas a 120K mentalement. Il compare a "ce que ca devrait couter". |
| Prix sweet spot | 15-30K EUR |
| Reduction reelle | 75-88% de reduction vs realite du marche |
| Pour le client | MEGA attractif |
| Pour toi | MEGA rentable (cout reel: environ 1K) |

### Etape 3 : Calculer TON ROI (ce que tu gagnes reellement)

**Pour chaque service, calculer :**

**Ton cout reel :**
- Temps humain : X heures x valeur heure interne
- Outils AI : environ 20-50 EUR (au prorata)
- Infrastructure : environ 10-20 EUR
- Total cout : rarement plus de 500-2000 EUR

**Ton revenu :** Prix facture en XXXX EUR

**Ta marge :** Marge = (Prix - Cout) / Prix. Objectif : superieur a 85% sur chaque service.

### Etape 4 : Valider avec le "Test du Oui Enthousiaste"

Quand tu annonces ton prix, observe la reaction :

| Reaction du client | Interpretation |
|--------------------|----------------|
| "Hmm, c'est un peu cher..." | Prix trop haut OU mauvais framing |
| "OK, ca me va" | Prix correct |
| "Ah vraiment? C'est tout?" | Prix TROP BAS, augmente |
| "Wow, c'est super interessant!" | SWEET SPOT parfait |

Tu veux la reaction 4 : enthousiaste mais pas choque.

## Les 5 Methodes de Pricing

### Methode 1 : Pricing par Valeur Creee (recommande pour 80% des cas)

**Formule : Prix = Valeur Marche x 25-40%**

**Exemple :** MVP SaaS qui couterait 100K en agence. Prix = 100K x 30% = 30K.

**Quand utiliser :**
- Projets development (MVP, apps, dashboards)
- Projets design (branding, UI)
- One-shot deliverables

### Methode 2 : Pricing par ROI Client (pour les clients orientes chiffres)

**Formule : Prix = ROI genere pour le client x 10-20%**

**Exemple 1 :** Tu construis un outil de sales automation. Le client estime gagner 200K/an de revenue additionnel. Prix = 200K x 15% = 30K.

**Exemple 2 :** Tu optimises leur funnel marketing. Le client estime gagner 50K/an en conversion. Prix = 50K x 20% = 10K.

**Quand utiliser :**
- Quand le ROI est quantifiable
- Outils de revenue (e-commerce, SaaS, lead gen)
- Optimisations mesurables

### Methode 3 : Pricing par Economie Realisee (pour les PME cost-conscious)

**Formule : Prix = Economie annuelle du client x 15-25%**

**Exemple :** Le client a une equipe de 3 devs a 200K/an. Ton systeme remplace 2 devs = economie 130K/an. Prix = 130K x 20% = 26K (one-shot) ou 2K/mois.

**Quand utiliser :**
- Transformation AI d'equipes existantes
- Remplacement de process manuels
- Clients qui veulent reduire les couts

### Methode 4 : Pricing par Bundle (pour maximiser le ticket moyen)

**Formule : Prix bundle = Somme des services individuels x 70-80%**

**Exemple :** Dev (20K) + Design (5K) + Marketing (3K) + PM (2K) = 30K. Bundle prix = 30K x 75% = 22.5K.

Le client percoit : "J'economise 7.5K en prenant le package." Toi : Tu gagnes un plus gros deal + engagement plus long.

**Quand utiliser :**
- Toujours proposer un bundle en alternative
- Clients qui ont besoin de plusieurs services
- Upsell naturel

### Methode 5 : Pricing par Retainer + Performance (pour le recurring)

**Formule : Prix = Base fixe mensuelle + Variable sur resultats**

**Exemples :**
- Marketing retainer : 2K/mois fixe + 10% des leads generes au-dessus de X
- CTO retainer : 3K/mois fixe + 5% equity
- Sales retainer : 1.5K/mois fixe + 5% du pipeline genere

**Quand utiliser :**
- Services ongoing (marketing, sales, support)
- Quand tu veux aligner tes interets avec le client
- Pour creer du revenue recurrent

## Calculateur de Prix (Decision Tree)

**Le client arrive avec un besoin. Suivre ces questions :**

**Q1 : Combien ca couterait avec l'alternative traditionnelle?** Reponse = VALEUR MARCHE

**Q2 : Quel ROI le client attend?**
- Reponse quantifiable : utiliser Methode 2 (ROI)
- Reponse vague : utiliser Methode 1 (Valeur Marche)

**Q3 : Le client est cost-conscious ou growth-focused?**
- Cost-conscious : Methode 3 (Economie), prix plus bas
- Growth-focused : Methode 2 (ROI), prix plus haut

**Q4 : Projet one-shot ou ongoing?**
- One-shot : Methode 1 ou 2
- Ongoing : Methode 5 (Retainer + Performance)

**Q5 : Plusieurs services necessaires?**
- Oui : Methode 4 (Bundle) avec reduction
- Non : Service individuel

**RESULTAT : TON PRIX**

**Validation finale :**
- Prix > 85% de marge? OK
- Prix < 50% du marche? OK (attractif pour le client)
- Les deux? SWEET SPOT, go.

## Exemples Concrets de Calcul

### Exemple 1 : Startup SaaS (pre-seed, 300K leves)

**Besoin :** MVP complet pour valider le marche

**Calcul :**
1. Valeur marche : Equipe de 3 devs, 4 mois = environ 100K EUR

3. Client = growth-focused (veut lancer vite)
4. One-shot avec possibilite de retainer

**Application Methode 1 (Valeur Marche) :** 100K x 25% = 25K EUR

**Application Methode 2 (ROI) :** S'ils atteignent 10K MRR en 6 mois = 120K/an. 120K x 15% = 18K EUR.

**Prix propose : 20K EUR** (entre les deux). Avec option equity 5% (-5K = 15K cash + 5% equity).

| Perspective | Detail |
|-------------|--------|
| Le client voit | "20K au lieu de 100K, en 4 semaines au lieu de 4 mois" = Decision facile |
| Cout reel | environ 1.5K (temps + outils) |

**Application Methode 3 (Economie) :** Economie : au moins 1 poste elimine = 40K/an. 40K x 25% = 10K EUR premiere annee.

**Prix propose : 12K EUR setup + 1K/mois maintenance**

| Perspective | Detail |

### Exemple 3 : Startup en croissance (veut scaler le marketing)

**Besoin :** Full marketing setup (content + SEO + social + email)

**Calcul :**
1. Valeur marche : Hire marketing manager = 55K/an + Agence content = 3K/mois = 36K/an. Total : environ 91K/an.
2. ROI attendu : Passer de 5K a 20K MRR = +180K/an
3. Client = growth-focused
4. Ongoing retainer

**Application Methode 5 (Retainer + Performance) :** Base : 3K/mois (vs 7.5K/mois pour un humain + agence). Variable : +5% du MRR au-dessus de 15K.

**Prix propose : 3K/mois + 5% du MRR > 15K + 3K setup**

| Si 50K MRR | 3K + 5% de 35K = 3K + 1.75K = 4.75K/mois |
| Resultat | Upside illimite, downside protege |

## Anti-Patterns de Pricing (ERREURS FATALES)

| Erreur | Pourquoi c'est fatal | Solution |
|--------|---------------------|----------|
| Facturer a l'heure | Tu te punis pour ta vitesse | Forfait base sur la valeur |
| Donner le prix avant le call | Le client juge sans contexte | "Ca depend du projet, parlons-en" |
| Negocier vers le bas | Devalue tout ton travail | Negocie le SCOPE, jamais le prix |
| Meme prix pour tous | Les clients ont des budgets differents | 3 options (Good/Better/Best) |
| Pas de prix affiche du tout | Clients non qualifies perdent du temps | Fourchette sur le site ("a partir de X") |
| Prix trop rond | "20 000 EUR" semble invente | "18 500 EUR" semble calcule |
| Pas d'ancrage | Le client compare a "rien" | Toujours montrer le prix du marche d'abord |

## Le "Pricing Presentation" (Comment Presenter)

### Structure de la presentation du prix

**ETAPE 1 : RAPPELER LA DOULEUR (30 sec)**

"Aujourd'hui, pour avoir ce produit, vous devriez embaucher 3-4 personnes pendant 6 mois. Ca represente environ 120K EUR et presque un an avant le premier utilisateur."

**ETAPE 2 : MONTRER LA VALEUR (30 sec)**

"Ce que je vous propose: un produit complet, teste, deploye, en 4 semaines. Avec mon systeme Agentik OS, vous avez la puissance d'une equipe de 10 pour le prix d'un."

**ETAPE 3 : PRESENTER LE PRIX (15 sec)**

"Pour ce projet, mon investissement est de 22K EUR. C'est 80% de moins que l'alternative, et 5x plus rapide."

**ETAPE 4 : SILENCE**

Laisse le client reagir. Ne parle pas.

**ETAPE 5 : OPTIONS (si besoin)**

"J'ai aussi une option a 15K avec un scope plus concentre sur les 3 features principales, et une option premium a 30K qui inclut le marketing et les 60 premiers jours de support."

### Le mot "investissement" pas "cout"

| Formulation | Effet |
|-------------|-------|
| "Ca coute 20K" | Depense, douleur |
| "L'investissement est de 20K" | Retour, valeur |
| "Pour 20K, vous recevez un produit qui en vaut 120K" | Cadrage complet (BEST) |
