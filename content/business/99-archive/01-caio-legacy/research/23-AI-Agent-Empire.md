## 04. What Is an AI Agent Swarm

```

  SINGLE AI ASSISTANT              AI AGENT SWARM
  ====================             ===============
  1 model, 1 task                  Multiple agents, coordinated
  Stateless (forgets)              Stateful (remembers context)
  Reactive (responds)              Proactive (plans and acts)
  Manual orchestration             Self-coordinating
  Limited tool use                 Rich tool ecosystem
  Linear capability                Emergent intelligence

  SWARM = Multiple specialized AI agents that:
  1. Have defined roles (marketer, coder, analyst, writer)
  2. Communicate with each other
  3. Share context and memory
  4. Coordinate on complex multi-step tasks
  5. Self-correct and iterate
  6. Operate with minimal human oversight

```

## 06. Claude Code Agent Teams

### The Native Multi-Agent Development System

Claude Opus 4.6 Agent Teams is a paradigm shift: multiple Claude instances
coordinate autonomously, challenge each other, and tackle complex codebases
in parallel.

```
  HOW IT WORKS
  ============

  TEAM LEAD (1 Claude session)
    |-- Coordinates all work
    |-- Assigns tasks to teammates
    |-- Synthesizes results
    |-- Makes architectural decisions

  TEAMMATE 1 (independent Claude session)
    |-- Works on assigned task
    |-- Has own context window
    |-- Can message team lead AND other teammates
    +-- Reports progress

  TEAMMATE 2 (independent Claude session)
    |-- Works on different task
    |-- Can discover issues and share with team
    +-- Reviews teammate 1's work

  TEAMMATE 3 (independent Claude session)
    |-- Runs tests on completed work
    +-- Reports bugs to team lead

  KEY DIFFERENCE FROM SUBAGENTS:
  - Subagents: can ONLY report to parent (no peer communication)
  - Agent Teams: teammates communicate DIRECTLY with each other
```

## 15. Pricing Models for Agent Swarm Services

### Five Core Pricing Strategies

```
  MODEL 1: OUTCOME-BASED
  =======================
  Charge based on results delivered.
  Example: "X% of cost savings" or "Y per qualified lead"
  Best for: Lead generation, cost reduction projects
  Risk: Attribution complexity
  Margin: Variable (can be very high)

  MODEL 2: AGENT-BASED SUBSCRIPTION
  ===================================
  Charge per agent type deployed.
  Example: "Basic package = content + social + analytics agents"
  Best for: Ongoing services with clear agent roles
  Risk: Clients may not understand "agents"
  Margin: 60-80%

  MODEL 3: USAGE-BASED (CONSUMPTION)
  ====================================
  Charge per unit (API call, minute, query, document).
  Example: "$0.50 per customer service conversation resolved"
  Best for: Variable workloads, pay-as-you-go clients
  Risk: Unpredictable revenue
  Margin: 50-70%

  MODEL 4: PROJECT-BASED + RETAINER
  ====================================
  One-time build fee + monthly maintenance.
  Example: "$10,000 setup + $2,000/month maintenance"
  Best for: Custom implementations
  Risk: Feast/famine if not enough retainers
  Margin: 60-70% on project, 80%+ on retainer

  MODEL 5: HYBRID (RECOMMENDED)
  ===============================
  Base subscription + usage tiers + outcome bonuses.
  Example: "$2,000/mo base + $50/meeting booked + $500 per deal closed"
  Best for: Enterprise clients who want aligned incentives
  Risk: Complex to manage
  Margin: 65-85%
```
