## Le Formulaire de Candidature

### Structure du Formulaire

| Section | Champs Cles | Strategie |
|---------|-------------|-----------|
| Fondateurs | Nom, background, LinkedIn | Background technique = essentiel |
| Entreprise | Description, URL, demo | Avoir un prototype FONCTIONNEL |
| Progres | Revenus, utilisateurs, croissance | Montrer de la traction meme petite |
| Idee | Probleme, solution, marche | Clarte > exhaustivite |
| Equite | Cap table, investisseurs | Clean cap table, pas trop dilue |
| Autres | Hacks, anecdotes | Personnalite, pas corporate |

### Ce qu'il Faut Ecrire (Champ par Champ)

**"Describe what your company does in 50 characters or less"**
```
MAUVAIS : "AI-powered life management platform with integrated health,
           finance, and career optimization using sovereign data"
BON :     "Your life, one AI. Your data, your rules."
```

**"What is your company going to make?"**
```
REGLE : 2 paragraphes max. Structure :
- Phrase 1 : Ce que c'est (concret, pas abstrait)
- Phrase 2 : Pour qui et pourquoi maintenant
- Phrase 3 : Comment ca marche (technique simple)
- Phrase 4 : Pourquoi c'est difficile a copier
```

**"Why did you pick this idea to work on?"**
```
REGLE : Personnel + insight unique
- Pas de "le marche est grand" (ils le savent)
- Montre que TU vis le probleme
- Montre un insight que les autres n'ont pas
- Montre que tu as deja commence (pas juste une idee)
```

**"What's new about what you're making?"**
```
REGLE : L'insight non-obvious
- Ce n'est PAS "on utilise l'IA" (tout le monde)
- C'est le POURQUOI ca n'existait pas avant
- C'est le changement technologique ou comportemental qui rend ca possible MAINTENANT
```

**"How far along are you?"**
```
REGLE : Faits > Plans
- "On a lance la beta il y a 3 semaines, 147 utilisateurs, 34% retention J7"
- PAS "On prevoit de lancer en Q2"
- Meme si c'est petit, du concret bat toujours du theorique
```

## La Video de 1 Minute

### Structure Optimale

```
+-- VIDEO YC : 60 SECONDES MAX ------------------------------------+
|                                                                    |
|  SECONDES 0-10 : QUI TU ES                                       |
|  |-- Prenom, background en 1 phrase                               |
|  |-- "Je suis [nom], j'ai [credential technique]"                |
|  +-- Pas de "bonjour", pas de plaisanteries                      |
|                                                                    |
|  SECONDES 10-25 : LE PROBLEME + SOLUTION                         |
|  |-- Le probleme en 1 phrase concrete                             |
|  |-- La solution en 1 phrase concrete                             |
|  +-- Pas de jargon, pas de buzzwords                              |
|                                                                    |
|  SECONDES 25-40 : DEMO                                            |
|  |-- Montrer le produit qui MARCHE                                |
|  |-- Pas de maquettes, pas de slides                              |
|  +-- L'ecran qui fait "wow"                                       |
|                                                                    |
|  SECONDES 40-50 : TRACTION / INSIGHT                              |
|  |-- Chiffres si tu en as                                         |
|  |-- Ou l'insight non-obvious qui fait tiquer                     |
|  +-- Pourquoi MAINTENANT                                          |
|                                                                    |
|  SECONDES 50-60 : AMBITION                                        |
|  |-- Ou ca va dans 5 ans                                          |
|  +-- Phrase finale memorable                                      |
|                                                                    |
+--------------------------------------------------------------------+
```

### Regles Techniques de la Video

| Element | Specification |
|---------|--------------|
| Duree | 60 secondes MAX (59 ideal) |
| Format | Face camera + screencast |
| Qualite | Webcam OK, pas besoin de studio |
| Eclairage | Lumiere naturelle, face visible |
| Son | Micro correct (pas d'echo) |
| Montage | Aucun effet, cuts simples |
| Energie | Parler vite mais clairement |
| Script | Ecrit et repete 20+ fois |

## L'Interview : 10 Minutes qui Changent Tout

### Format

| Parametre | Detail |
|-----------|--------|
| Duree | 10 minutes exactement |
| Nombre de partners | 2-3 |
| Format | Visio (Zoom) |
| Style | Questions rapides, interruptions frequentes |
| Objectif des partners | Comprendre si tu COMPRENDS ton probleme |

### Les 12 Questions les Plus Frequentes

| # | Question | Ce qu'ils evaluent |
|---|----------|---------------------|
| 1 | "What does your company do?" | Clarte, simplicite |
| 2 | "Why are you the right team?" | Founder-market fit |
| 3 | "How do you make money?" | Sens business |
| 4 | "What's your unfair advantage?" | Defensibilite |
| 5 | "Who are your competitors?" | Connaissance du marche |
| 6 | "What's the biggest risk?" | Honnetete, lucidite |
| 7 | "How did you get your first users?" | Resourcefulness |
| 8 | "What do you know that others don't?" | Insight unique |
| 9 | "Why hasn't this been built before?" | Timing |
| 10 | "What's your growth rate?" | Traction |
| 11 | "What will you do with the money?" | Priorites |
| 12 | "Where do you see this in 5 years?" | Ambition |

### Comment Repondre

```
REGLE D'OR : REPONSES DE 15-30 SECONDES MAX

STRUCTURE PAR REPONSE :
1. Reponse directe (1 phrase)
2. Preuve ou exemple (1 phrase)
3. Stop. Attendre la question suivante.

INTERDIT :
- Reponses de plus de 45 secondes
- "C'est une bonne question" (perte de temps)
- Eviter une question (ils insisteront)
- Jargon technique excessif
- Dire "on va faire" au lieu de "on a fait"
```

### Preparation Interview

| Action | Detail |
|--------|--------|
| Mock interviews | 10 minimum avec d'autres founders |
| Timer | Repeter avec chrono, 15 sec/reponse |
| Co-fondateur | Si tu es 2, diviser les reponses par expertise |
| Anticipation | Preparer les 3 questions les plus dures |
| Energie | Cafe, pas de meeting avant, etre SHARP |

## Le Fondateur Solo chez YC

### Statistiques

| Fait | Chiffre |
|------|---------|
| Solo founders acceptes | ~15-20% des batches |
| Taux de reussite post-YC (solo) | Legerement inferieur mais viable |
| Exemples celebres | Sam Altman a pousse pour, Drew Houston (Dropbox early) |

### Comment Compenser le "Solo Disadvantage"

```
1. VITESSE D'EXECUTION
   |-- Montre que tu as deja construit le MVP seul
   |-- Deploiement rapide = preuve que tu n'as pas besoin d'equipe (encore)
   +-- Timeline de ce que tu as fait en X semaines

2. BACKGROUND TECHNIQUE FORT
   |-- Fondateur qui code > fondateur qui manage
   |-- Portfolio de projets techniques
   +-- Open source contributions si possible

3. AI LEVERAGE
   |-- "Je suis 1 personne mais je ship comme 5"
   |-- Montrer comment tu utilises l'IA pour multiplier ton output
   +-- C'est le narratif parfait pour 2026

4. PLAN DE RECRUTEMENT
   |-- "Je cherche un co-fondateur en [X]"
   |-- "YC m'aidera a trouver la bonne personne"
   +-- Montrer que tu sais ce qui te manque
```
