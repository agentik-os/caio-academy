# Marketplace d'Agents IA: L'App Store de 2027

> Construire une marketplace ou les gens achetent, vendent et louent des agents IA et des workflows. Revenue: 15-30% de commission. Approche verticale d'abord.

---

## Le Concept

Le GPT Store d'OpenAI a echoue parce qu'il est trop generique, sans controle qualite, et sans monetisation reelle pour les createurs. L'opportunite: construire la VRAIE marketplace d'agents IA -- avec curation, qualite, et un vrai business model. Pense Fiverr (services) + Shopify App Store (plugins) + Product Hunt (decouverte), mais pour des agents IA qui executent des taches reelles.

## Architecture Technique

### Stack

| Couche | Technologie | Justification |
|--------|------------|---------------|
| Frontend | Next.js 15 + Tailwind | SEO, SSR, rapide |
| Backend | Next.js API + Convex | Realtime, serverless |
| Auth | Clerk (multi-role: buyer/seller/admin) | Roles natifs |
| Paiement | Stripe Connect | Marketplace split payments |
| Agent Runtime | Docker containers + API proxy | Isolation par agent |
| Search | Algolia ou Meilisearch | Recherche instantanee |
| Reviews | Custom (Convex) | Systeme de notation |
| Analytics | PostHog | Tracking comportement |
| CDN | Cloudflare | Performance globale |

### Flow d'Utilisation

```
+------------------------------------------------------------------+
|              PARCOURS UTILISATEUR                                  |
+------------------------------------------------------------------+
|                                                                    |
|  ACHETEUR:                                                        |
|  1. Browse / Search agents par categorie                          |
|  2. Voir la fiche: description, demo, reviews, prix              |
|  3. "Try Free" (essai 7 jours) ou "Buy Now"                      |
|  4. Connecter ses outils (Slack, CRM, email)                     |
|  5. L'agent travaille en background                               |
|  6. Dashboard de resultats + metriques                            |
|  7. Laisser une review                                            |
|                                                                    |
|  VENDEUR:                                                         |
|  1. Creer un compte vendeur (gratuit)                             |
|  2. Uploader son agent (code ou no-code)                          |
|  3. Ecrire la fiche produit (description, demo, pricing)          |
|  4. Soumettre pour review (qualite)                               |
|  5. Agent publie sur la marketplace                               |
|  6. Recevoir les paiements (Stripe Connect)                       |
|  7. Dashboard analytics: ventes, churn, revenus                   |
+------------------------------------------------------------------+
```

## Risques et Mitigations

| Risque | Probabilite | Impact | Mitigation |
|--------|------------|--------|------------|
| Pas assez de supply (createurs) | Elevee | Critique | Builder 30 agents soi-meme, programme founding creator |
| Pas assez de demand (acheteurs) | Elevee | Critique | Content marketing massif, niche verticale d'abord |
| Qualite des agents mediocre | Moyenne | Fort | Review process strict, systeme de rating, garantie |
| Competition de OpenAI/Anthropic | Moyenne | Critique | Curation + vertical = ce qu'ils ne font pas |
| Problemes techniques (runtime agents) | Moyenne | Fort | Sandboxing Docker, monitoring, SLA |
| Fraude (vendeurs malveillants) | Faible | Moyen | Verification identite, escrow payments |
| Unit economics negatives | Moyenne | Fort | Commission assez elevee (20%+), couts infra controles |
