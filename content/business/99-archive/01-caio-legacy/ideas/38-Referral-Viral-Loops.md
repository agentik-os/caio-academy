# Boucles Virales: Chaque Client Amene le Suivant

> Guide complet pour integrer la viralite dans un produit IA.
> Objectif: K-factor > 0.3 (chaque 10 utilisateurs en amenent 3+).

---

## Les 4 Types de Viralite

### Classification

| Type | Mecanisme | Exemple | K-factor Typique |
|------|-----------|---------|-----------------|
| Word-of-Mouth | Le produit est tellement bon qu'on en parle | ChatGPT, Notion | 0.1-0.3 |
| Incentivized Referral | Recompense pour inviter | Dropbox, Revolut | 0.3-0.8 |
| Built-in (Output Sharing) | L'usage genere de la visibilite | Canva ("Made with Canva"), Loom | 0.2-0.5 |
| Network Effects | Plus d'users = plus de valeur | WhatsApp, Slack, LinkedIn | 0.5-2.0+ |

### Quel Type Choisir pour un Produit IA

```
PRODUIT IA (SaaS B2B): Output Sharing + Referral
  - L'agent genere un rapport -> le client le partage -> visibilite
  - Le client invite un collegue -> les deux gagnent

COMMUNAUTE IA: Network Effects + Referral
  - Plus de membres = plus de valeur (discussions, contacts)
  - Parrainage = mois gratuit

OUTIL IA (B2C): Built-in + Word-of-Mouth
  - Watermark/credit sur les outputs
  - Tellement utile que les gens en parlent

AGENCE IA: Word-of-Mouth + Referral
  - La qualite du travail genere des recommandations
  - Programme de referral: 10% commission
```

## Output Sharing: La Viralite Invisible

### Le Concept

```
CHAQUE FOIS QU'UN UTILISATEUR UTILISE TON PRODUIT IA,
LE RESULTAT GENERE CONTIENT UNE TRACE DE TON BRAND.

Exemples:
  - Rapport IA: "Genere par AgentikOS" + lien en footer
  - Dashboard: URL partageable avec branding
  - Email automatise: "Powered by [Product]" en signature
  - PDF genere: Logo + lien sur chaque page
  - Chatbot: "Propulse par [Product]" dans le widget
```

### Implementation pour Differents Produits IA

| Produit | Output | Attribution |
|---------|--------|-------------|
| Agent de rapport | PDF/HTML genere | Footer: "Cree avec AgentikOS - agentik-os.com" |
| Agent de qualification leads | Email de suivi | Signature: "Qualification automatisee par [Product]" |
| Dashboard IA | URL partageable | Barre superieure: "Powered by AgentikOS" |
| Chatbot client | Widget web | Badge: "IA par AgentikOS" |
| Agent SEO | Audit HTML | Header + footer du rapport |
| Agent de code | README genere | Commentaire: "Generated by [Product]" |

### Metriques Output Sharing

| Metrique | Definition | Benchmark |
|----------|-----------|-----------|
| Share Rate | % d'outputs partages en dehors du produit | 5-15% |
| Attribution Click Rate | % de personnes qui cliquent sur le lien | 1-5% des viewers |
| Signup from Share | % qui s'inscrivent apres clic | 10-30% des cliqueurs |
| K-factor (output) | Share Rate x Click Rate x Signup Rate | 0.05-0.2 |

## Referral Program: Le Programme de Parrainage

### Architecture du Programme

```
MECANIQUE DE BASE:
  - Le parrain invite -> le filleul s'inscrit
  - Le parrain recoit: [recompense A]
  - Le filleul recoit: [recompense B]
  - Les deux sont gagnants (bilateral)

RECOMPENSES RECOMMANDEES:
  Option 1: Credit produit
    Parrain: 1 mois gratuit
    Filleul: 1 mois gratuit
    [Meilleur pour SaaS avec abonnement]

  Option 2: Cash/credit
    Parrain: 20 EUR de credit
    Filleul: 20 EUR de credit
    [Meilleur pour produits avec usage variable]

  Option 3: Feature unlock
    Parrain: Acces a une feature premium
    Filleul: Essai premium 14 jours
    [Meilleur pour freemium]

  Option 4: Commission recurrente
    Parrain: 20% de commission sur chaque paiement du filleul
    Filleul: 10% de reduction permanent
    [Meilleur pour high-ticket services]
```

### Design du Referral Flow

| Etape | Ecran/Action | Objectif |
|-------|-------------|----------|
| 1. Trigger | Moment de satisfaction (apres resultat positif) | Capter l'emotion positive |
| 2. Invite | Page dediee avec lien unique + boutons partage | Faciliter l'action |
| 3. Landing | Page d'accueil specifique filleul (mentionne le parrain) | Confiance + conversion |
| 4. Signup | Inscription avec credit automatique | Friction zero |
| 5. Notification | Email/notification au parrain: "X s'est inscrit!" | Dopamine + engagement |
| 6. Reward | Credit/recompense automatique | Renforcement du comportement |

### Quand Declencher le Referral

| Moment | Taux de Conversion | Justification |
|--------|-------------------|---------------|
| Apres un resultat positif (rapport genere, lead qualifie) | Eleve (15-25%) | Satisfaction maximale |
| Apres 7 jours d'utilisation | Moyen (8-15%) | Habitude formee |
| Apres un milestone (10 rapports, 50 leads) | Eleve (12-20%) | Sentiment d'accomplissement |
| Email mensuel "Invitez vos collegues" | Faible (2-5%) | Rappel passif |
| In-app prompt apres login | Moyen (5-10%) | Visibilite constante |

## Metriques de Viralite

### Les Formules Cles

```
K-FACTOR (Coefficient Viral):
  K = nombre d'invitations par utilisateur x taux de conversion des invitations

  Exemple:
    Chaque utilisateur invite en moyenne 3 personnes
    20% de ces invitations se convertissent en utilisateurs
    K = 3 x 0.20 = 0.6

  Interpretation:
    K < 0.1 = pas de viralite
    K = 0.1-0.3 = viralite legere (aide a la croissance)
    K = 0.3-0.7 = bonne viralite (reduit le CAC significativement)
    K = 0.7-1.0 = forte viralite (croissance quasi-organique)
    K > 1.0 = VIRALE (croissance exponentielle -- rare et temporaire)

CYCLE TIME (Temps du Cycle Viral):
  Temps entre: un utilisateur s'inscrit -> son filleul s'inscrit

  Benchmark:
    < 3 jours = viral explosif
    3-7 jours = excellent
    7-14 jours = bon
    14-30 jours = correct
    > 30 jours = trop lent

  REGLE: K-factor eleve + cycle time court = croissance exponentielle
```

### Dashboard de Suivi Viralite

| Metrique | Definition | Objectif |
|----------|-----------|----------|
| K-factor global | Invitations x conversion | > 0.3 |
| Invitations par user | Nombre moyen d'invitations envoyees | > 2 |
| Conversion invitation | % d'invites qui s'inscrivent | > 15% |
| Cycle time | Jours entre inscription et premier filleul | < 7 jours |
| Share rate (output) | % d'outputs partages | > 10% |
| Organic signups | % de signups sans source payante | > 40% |
| NPS (Net Promoter Score) | Probabilite de recommandation | > 50 |

## Network Effects pour Produits IA

### Types de Network Effects Applicables

| Type | Application IA | Exemple |
|------|---------------|---------|
| Direct | Plus d'utilisateurs = meilleurs modeles (data) | L'agent apprend des usages collectifs |
| Cross-side | Marketplace: plus de freelances IA + plus de clients | Plateforme de matching |
| Data | Plus de donnees = predictions meilleures | Recommendations personnalisees |
| Contenu | Plus de membres = plus de discussions/ressources | Communaute IA |

### Implementation pour une Communaute IA

```
NETWORK EFFECTS DANS UNE COMMUNAUTE PAYANTE:

  10 membres: Peu de valeur, contenu du createur seulement
  50 membres: Discussions actives, premiers networking
  100 membres: Templates partages, collabs entre membres
  300 membres: Deals exclusifs, job board actif
  500 membres: Expert directory, sous-groupes specialises
  1000 membres: Ecosystem complet, la communaute est auto-suffisante

  POINT DE BASCULE: ~200-300 membres
  Avant: la valeur vient du createur
  Apres: la valeur vient des membres eux-memes
  -> Le churn chute drastiquement apres ce point
```

## Implementation Technique

### Outils de Referral

| Outil | Prix | Ideal Pour | Features |
|-------|------|-----------|----------|
| Rewardful | 49-299 USD/mois | SaaS + Stripe | Affilies + referral, dashboard |
| Viral Loops | 49-299 USD/mois | Pre-launch + waitlists | Templates viraux, leaderboard |
| ReferralCandy | 59-299 USD/mois | E-commerce | Automatisation complete |
| Tolt | 29 USD/mois | SaaS bootstrapped | Simple, Stripe natif |
| Custom (base de donnees) | 0 (dev time) | Controle total | Flexibilite maximale |

### Implementation Custom (Recommandee pour Controle)

```
SCHEMA BASE DE DONNEES:

  users:
    id, email, referral_code (unique), referred_by

  referrals:
    id, referrer_id, referred_id, status, reward_given, created_at

  referral_rewards:
    id, user_id, type (credit/month_free/cash), amount, claimed_at

ENDPOINTS API:
  GET  /api/referral/code      -> Retourne le code unique du user
  GET  /api/referral/stats     -> Stats: invites, convertis, rewards
  POST /api/referral/track     -> Enregistre un clic sur lien referral
  POST /api/referral/claim     -> Reclame une recompense

FLOW:
  1. User partage: https://app.com?ref=ABC123
  2. Visiteur clique -> cookie stocke "ref=ABC123"
  3. Visiteur s'inscrit -> referral tracke
  4. Apres paiement -> recompenses distribuees aux deux parties
```

## Playbook: 90 Jours pour Activer la Viralite

### Phase 1: Foundation (Jours 1-30)

| Semaine | Actions |
|---------|---------|
| S1 | Ajouter attribution "Powered by [Product]" sur tous les outputs |
| S2 | Creer la page referral dans l'app (lien unique + stats) |
| S3 | Implementer le tracking (cookies, base de donnees, analytics) |
| S4 | Lancer le programme avec les utilisateurs existants |

### Phase 2: Optimisation (Jours 31-60)

| Semaine | Actions |
|---------|---------|
| S5 | Analyser les premieres donnees: qui partage? quand? comment? |
| S6 | A/B tester les recompenses (credit vs mois gratuit vs feature) |
| S7 | Optimiser les triggers (quand demander le referral) |
| S8 | Ajouter les boutons de partage social (Twitter, LinkedIn, email) |

### Phase 3: Scale (Jours 61-90)

| Semaine | Actions |
|---------|---------|
| S9 | Leaderboard des meilleurs referrers (gamification) |
| S10 | Programme ambassadeur (top 10% referrers = avantages exclusifs) |
| S11 | Campagne email dediee: "Invitez 3 collegues, recevez [X]" |
| S12 | Analyser K-factor et cycle time. Iterer sur les points faibles. |

### Objectifs a 90 Jours

| Metrique | Objectif Jour 30 | Objectif Jour 60 | Objectif Jour 90 |
|----------|------------------|-------------------|-------------------|
| K-factor | 0.1 | 0.2 | 0.3+ |
| Share rate | 5% | 8% | 12%+ |
| Referral signups | 5% des signups | 10% des signups | 20% des signups |
| Cycle time | 14 jours | 10 jours | 7 jours |

## Erreurs a Eviter

| Erreur | Consequence | Solution |
|--------|------------|----------|
| Recompense trop faible | Personne ne partage | Minimum 1 mois gratuit ou equivalent |
| Recompense unilaterale | Filleul n'a pas d'incentive | Toujours bilateral (les deux gagnent) |
| Referral cache dans les settings | Personne ne le trouve | Bouton visible dans le dashboard principal |
| Pas de tracking | Impossible d'optimiser | Analytics des le jour 1 |
| Demander le referral trop tot | User pas encore convaincu | Attendre un moment de satisfaction |
| Output sans attribution | Viralite perdue | "Powered by" sur TOUT ce que le produit genere |
| Programme trop complexe | Confusion, abandon | 1 action, 1 recompense, c'est tout |

## Calcul d'Impact sur le CAC

```
SANS VIRALITE:
  CAC (cout d'acquisition) = 50 EUR
  1 000 clients = 50 000 EUR de marketing

AVEC K-FACTOR DE 0.3:
  Pour 1 000 clients, il faut en acquérir: 1000 / (1 + 0.3 + 0.09 + ...) = ~770
  770 x 50 EUR = 38 500 EUR
  Economie: 11 500 EUR (23%)

AVEC K-FACTOR DE 0.6:
  Pour 1 000 clients, il faut en acquérir: ~625
  625 x 50 EUR = 31 250 EUR
  Economie: 18 750 EUR (37.5%)

AVEC K-FACTOR DE 1.0+:
  Croissance exponentielle: chaque utilisateur en amene 1+
  Le budget marketing peut etre reinvesti en produit
  -> C'est le reve de toute startup
```
