## Le Concept

Le white-label est le modele ultime du "build once, sell many". Tu construis une plateforme IA complete, puis tu la vends sous forme de licence mensuelle a des agences et revendeurs qui la personnalisent avec leur propre marque. Ils facturent leurs clients 3-10x ce qu'ils te paient. Tout le monde gagne.

C'est le modele de GoHighLevel (valorise a 1 Mrd USD), de Vendasta, de DashClicks. La difference: toi, tu le fais avec l'IA native, pas du CRM recycle.

## Architecture Technique du Produit

### Stack Recommande

| Couche | Technologie | Justification |
|--------|------------|---------------|
| Frontend | Next.js 15 + Tailwind | Multi-tenant, SSR, rapide |
| Backend | Next.js API + tRPC | Type-safe, monorepo |
| Database | PostgreSQL (Supabase) | Multi-tenant par schema/RLS |
| Auth | Clerk (organizations) | Multi-org natif |
| IA | Claude API + fallback GPT-4 | Fiabilite + qualite |
| File Storage | S3/R2 (Cloudflare) | Pas cher, scalable |
| Billing | Stripe Billing | Subscriptions, metering |
| Realtime | Pusher ou Supabase RT | Chat, notifications |
| Deployment | Vercel | Auto-scale, zero config |

### Architecture Multi-Tenant

```
+------------------------------------------------------------------+
|                    ARCHITECTURE MULTI-TENANT                      |
+------------------------------------------------------------------+
|                                                                    |
|  [Plateforme Core]                                                |
|       |                                                            |
|       +-- [Tenant Layer]                                          |
|       |   |-- tenant_id (chaque agence)                           |
|       |   |-- custom_domain                                       |
|       |   |-- brand_config (logo, couleurs, nom)                  |
|       |   |-- api_keys                                            |
|       |   +-- sub_accounts (clients de l'agence)                  |
|       |                                                            |
|       +-- [Feature Modules]                                       |
|       |   |-- Chatbot Engine                                      |
|       |   |-- Content Generator                                   |
|       |   |-- Analytics Dashboard                                 |
|       |   |-- Email Automations                                   |
|       |   +-- Reporting Engine                                    |
|       |                                                            |
|       +-- [Infrastructure]                                        |
|           |-- Rate Limiting (par tenant)                           |
|           |-- Usage Metering (tokens IA)                           |
|           |-- Billing (Stripe per-tenant)                          |
|           +-- Audit Logs                                          |
+------------------------------------------------------------------+
```

## Risques et Mitigations

| Risque | Probabilite | Impact | Mitigation |
|--------|------------|--------|------------|
| Complexite technique multi-tenant | Elevee | Critique | Commencer simple (1 schema), evoluer |
| GoHighLevel ajoute des features IA | Elevee | Fort | Se specialiser IA-native, pas CRM generique |
| Churn eleve des agences | Moyenne | Fort | Stickiness: les clients finaux sont sur la plateforme |
| Support overwhelmant | Moyenne | Moyen | Documentation, videos, community forum |
| Cout API IA imprevisible | Moyenne | Moyen | Usage-based pricing, limites par tier |
| Dependance a 1-2 gros clients | Faible | Fort | Diversifier, pas de client >10% du MRR |
| Concurrence de nouveaux entrants | Elevee | Moyen | Avantage premier entrant, effets de reseau |
