## Structure Complète des Modules

### MODULE 1 — Le Nouveau Paysage de la Recherche en 2026

**Objectif** : L'eleve comprend comment la recherche a fondamentalement change et ce que ca implique pour sa stratégie.

**Lecons a générer :**

1.1 **La recherche, c'est plus seulement Google**
- Google domine toujours (~85% de part de marche) MAIS l'IA grignote du terrain
- Nouvelles surfaces de recherche : Google AI Overviews, ChatGPT web search, Perplexity, Bing Copilot, You.com
- Zero-click searches : 60%+ des recherches Google ne generent aucun clic
- L'economie de la réponse : les gens veulent des réponses, pas des liens
- Ton contenu doit fonctionner pour la recherche traditionnelle ET la recherche IA

1.2 **Comment Google AI Overviews change le SEO**
- Google synthetise maintenant des réponses depuis plusieurs sources en haut des resultats
- Être cite dans les AI Overviews = visibilite massive (position 0+)
- Ce qui est cite : du contenu structure, factuel, bien source
- Ce qui est ignore : du contenu generique, superficiel, de l'opinion sans preuves
- Le nouveau jeu : optimiser pour la citabilite, pas juste les rankings

1.3 **Comment fonctionnent les moteurs de recherche IA (ChatGPT, Perplexity, etc.)**
- Ils crawlent, synthetisent et citent — différent des 10 liens bleus de Google
- Être cite = ta marque mentionnee dans les réponses générées par l'IA
- AI crawlers : GPTBot, ClaudeBot, PerplexityBot, etc.
- Stratégie robots.txt : AUTORISER tous les AI crawlers (ne bloque pas ta source de trafic)
- llms.txt : le nouveau standard pour les resumes de sites lisibles par l'IA

1.4 **Le framework de double optimisation**
- SEO (Search Engine Optimization) : se classer dans les resultats traditionnels de Google
- GEO (Generative Engine Optimization) : être cite dans les réponses générées par l'IA
- Tu as besoin des DEUX — ils sont complementaires, pas en competition
- La bonne nouvelle : 80% des bonnes pratiques SEO aident aussi le GEO

### MODULE 2 — Les Fondations du SEO Technique

**Objectif** : L'eleve s'assure que son site est techniquement solide pour les moteurs de recherche et les AI crawlers.

**Lecons a générer :**

2.1 **Architecture de site pour le SEO**
- Structure des URL : propres, descriptives, riches en mots-cles
- Hierarchie : accueil → categorie → page (max 3 clics de profondeur)
- Maillage interne : chaque page doit être accessible depuis au moins 2 autres pages
- Breadcrumbs : aident les utilisateurs et les moteurs de recherche a comprendre la structure
- Navigation : claire, crawlable, mobile-friendly

2.2 **Core Web Vitals (les metriques de performance que Google surveille)**
- LCP (Largest Contentful Paint) : contenu principal visible en < 2.5s
- INP (Interaction to Next Paint) : réponse a l'input en < 200ms
- CLS (Cumulative Layout Shift) : stabilite visuelle < 0.1
- Comment mesurer : PageSpeed Insights, Lighthouse, extension Web Vitals
- Corrections courantes : optimisation des images, chargement des polices, report du JavaScript

2.3 **Crawlabilite et indexabilite**
- robots.txt : quoi autoriser, quoi bloquer
- Sitemap.xml : doit inclure toutes les pages importantes, exclure les pages faibles
- URL canoniques : prevenir les problèmes de contenu duplique
- noindex/nofollow : quand les utiliser (pages faibles, pages admin)
- Gestion des redirections : 301 pour permanent, 308 pour permanent moderne
- Headers HTTP : X-Robots-Tag, rel=canonical

2.4 **Donnees structurees (Schema.org)**
- Ce que c'est : des metadonnees lisibles par les machines sur ton contenu
- Types de schema cles : Organization, WebPage, FAQPage, BreadcrumbList, Article, Product, LocalBusiness, Service
- Implementation : JSON-LD dans le `<head>` (le format recommande)
- Validation : Google Rich Results Test, validateur Schema.org
- Impact : rich snippets, knowledge panels, citations IA

2.5 **Optimisation pour les AI crawlers**
- Autoriser tous les AI crawlers dans robots.txt (GPTBot, ClaudeBot, PerplexityBot, etc.)
- Créer `/llms.txt` et `/llms-full.txt` pour les resumes de site lisibles par l'IA
- Rendre le contenu côté serveur (pas de rendu client-side uniquement pour le texte important)
- Les donnees structurees aident l'IA a comprendre ton contenu
- Les sections FAQ sont de l'or pour les citations IA

2.6 **Mobile-first indexing**
- Google indexe la version mobile en premier
- Le design responsive est non-negociable
- Test : Google Mobile-Friendly Test
- Problèmes courants : texte trop petit, boutons non-cliquables, scroll horizontal

**Exercice** : Lance un audit SEO technique complet sur ton site. Corrige tous les problèmes critiques.

### MODULE 3 — Recherche de Mots-Cles & Stratégie de Contenu avec l'IA

**Objectif** : L'eleve sait trouver des mots-cles a forte valeur et construire une stratégie de contenu avec des outils IA.

3.3 **Calendrier de contenu avec l'IA**
- Utiliser Claude pour générer 3 mois d'idees de contenu
- Priorisation : volume de recherche x intention x competition x pertinence
- Rythme de publication : la regularite > la frequence
- Sujets saisonniers et tendances

3.4 **Analyse du contenu concurrent**
- Reverse-engineer les pages les mieux classees
- Ce qu'ils couvrent et pas toi
- Analyse des formats de contenu (listes, guides, comparaisons, outils)
- Analyse des SERP features : ce qui apparait dans Google pour tes mots-cles

### MODULE 4 — Création de Contenu IA a Grande Echelle

**Objectif** : L'eleve maitrise la création de contenu assistee par l'IA qui se classe et convertit.

**Lecons a générer :**

4.1 **Le framework de création de contenu IA**
- L'IA ecrit le premier brouillon (80% du travail)
- L'humain ajoute l'expertise, l'expérience et l'opinion (E-E-A-T)
- L'IA optimise pour le SEO (meta tags, structure, mots-cles)
- L'humain revoit et publie
- Ce n'est PAS "laisser l'IA tout ecrire" — c'est l'expertise humaine augmentee par l'IA

4.2 **Ecrire des articles SEO-optimises avec Claude**
- Prompt engineering pour le contenu SEO
- Structure d'article : réponse d'abord, puis la profondeur
- Hierarchie de titres : H1 → H2 → H3 (logique, riche en mots-cles)
- Nombre de mots : viser 2000-3000 mots pour les sujets competitifs
- Integration naturelle des mots-cles (pas de bourrage)
- Maillage interne et liens externes
- Sections FAQ (excellentes pour les featured snippets + citations IA)

4.3 **Les formats de contenu qui se classent**

| Format | Ideal pour | Exemple |
|--------|------------|---------|
| **Guide ultime** | Mots-cles haute competition | "Guide complet du SEO IA 2026" |
| **Comparaison** | Intention commerciale | "Claude vs ChatGPT pour le business" |
| **Tutoriel** | Intention informationnelle | "Comment configurer des serveurs MCP" |
| **Liste** | Sujets larges | "15 meilleurs outils IA pour le marketing" |
| **Etude de cas** | Construction de confiance | "Comment on a augmente le trafic de 300%" |
| **Glossaire** | Mots-cles longue traine | "Qu'est-ce que l'Agentic AI ?" |
| **Page outil** | Intention commerciale | "Outil d'audit IA gratuit" |

4.4 **E-E-A-T : Ce que Google veut vraiment**
- Expérience : montre que tu as FAIT ce dont tu parles
- Expertise : demontre une connaissance approfondie
- Autorite : sois reconnu dans ton domaine
- Fiabilite : sois précis, cite tes sources, sois transparent
- Comment signaler l'E-E-A-T dans ton contenu (bios d'auteurs, etudes de cas, donnees, citations)

4.5 **Eviter les penalites de detection de contenu IA**
- Google ne penalise PAS le contenu IA — ils penalisent le contenu de BASSE Qualité
- Ajoute de l'expérience personnelle, des donnees uniques, des insights originaux
- Ne publie pas le brut de l'IA sans relecture humaine
- Varie la structure et la voix de ton contenu
- Inclus des images originales, graphiques et donnees

4.6 **Pipelines de contenu automatises**
- L'approche Agentik OS : cron → Claude généré → optimisation SEO → génération d'images → build → deploy → publication sociale → notification
- Configurer l'auto-publishing avec les skills Claude Code
- Quality gates : relecture avant publication
- Passage a l'echelle : 1 article/jour en autopilote

### MODULE 5 — GEO : Generative Engine Optimization

**Objectif** : L'eleve optimise son contenu pour les moteurs de recherche IA.

**Lecons a générer :**

5.1 **C'est quoi le GEO et pourquoi c'est important**
- GEO = rendre ton contenu citable par les moteurs de recherche IA
- Les moteurs IA citent du contenu qui est : factuel, structure, bien source, definitif
- Être cite dans ChatGPT/Perplexity/AI Overviews = visibilite de marque a grande echelle
- C'est l'equivalent SEO de la "position 0" mais pour l'IA

5.2 **Les 7 signaux GEO**

| Signal | Ce que ca veut dire | Comment optimiser |
|--------|---------------------|-------------------|
| **Citabilite** | L'IA peut-elle citer ton contenu directement ? | Ecris des phrases claires et factuelles qui se suffisent a elles-mêmes |
| **Structure** | Ton contenu est-il bien organise ? | Utilise des titres, listes, tableaux, sections FAQ |
| **Autorite** | Ta source est-elle fiable ? | Construis des backlinks, citations, E-E-A-T |
| **Fraicheur** | Le contenu est-il a jour ? | Mets a jour regulierement, inclus des dates |
| **Completude** | Le sujet est-il couvert en profondeur ? | Sois la source la plus complète |
| **Unicite** | Offres-tu des insights uniques ? | Ajoute des donnees originales, etudes de cas, expérience |
| **Accessibilite** | Les AI crawlers peuvent-ils y acceder ? | Autorise les bots IA, utilise le server-rendering |

5.3 **llms.txt : Ton resume lisible par l'IA**
- Ce que c'est : un fichier texte a `/llms.txt` qui resume ton site pour les AI crawlers
- Format : markdown structure avec les informations cles
- `/llms.txt` (bref) et `/llms-full.txt` (complet)
- Quoi inclure : infos sur l'entreprise, services, contenu cle, donnees structurees

5.4 **Citabilite au niveau du passage**
- Les moteurs IA citent des PASSAGES, pas des pages
- Ecris des paragraphes autonomes qui repondent a des questions specifiques
- Chaque paragraphe doit être independamment significatif
- Utilise le "test Wikipedia" : ce paragraphe pourrait-il être cite dans une encyclopedie ?

5.5 **Optimisation FAQ pour l'IA**
- Les sections FAQ sont la source #1 pour les citations IA
- Chaque Q&R doit être autonome (question + réponse complète)
- Utilise les vraies questions que les gens cherchent
- Inclus des chiffres, dates et faits specifiques
- Balisage schema (FAQPage) pour le double benefice : Google + moteurs IA

5.6 **Optimisation par plateforme**
- Google AI Overviews : donnees structurees, E-E-A-T, contenu complet
- ChatGPT : faits clairs, passages citables, sources faisant autorite
- Perplexity : contenu recent, citations solides, réponses completes
- Bing Copilot : balisage Schema.org, optimisation Bing Webmaster

**Exercice** : Audite 5 pages pour les signaux GEO. Optimise-les pour la citabilite IA.

### MODULE 6 — SEO Programmatique a Grande Echelle

**Objectif** : L'eleve sait générer des centaines de pages SEO-optimisees a partir de templates et de donnees.

**Lecons a générer :**

6.1 **C'est quoi le SEO programmatique (pSEO)**
- Générer des centaines ou milliers de pages a partir d'un template + donnees
- Exemple : Zapier = 16.2M de visiteurs mensuels depuis 1.3M de mots-cles via pSEO
- Pattern : "[Outil A] + [Outil B] integration" x 1000 combinaisons
- Ce n'est PAS du spam — chaque page doit apporter une vraie valeur unique

6.2 **Les types de pages pSEO qui marchent**

| Type | Template | Exemple |
|------|----------|---------|
| **Comparaison** | "[Produit] vs [Produit]" | "Claude Code vs Cursor" |
| **Integration** | "[Outil A] + [Outil B]" | "Slack + GitHub Integration" |
| **Localisation** | "[Service] a [Ville]" | "Consulting IA a Paris" |
| **Cas d'usage** | "[Service] pour [Industrie]" | "Agents IA pour la Sante" |
| **Glossaire** | "Definition de [Terme]" | "C'est quoi l'Agentic AI ?" |
| **Alternative** | "Alternatives a [Produit]" | "Alternatives a Zapier 2026" |

6.3 **Construire du pSEO avec Next.js**
- Routes dynamiques : `[slug]/page.tsx` avec `generateStaticParams()`
- Fichiers data : JSON/TypeScript avec du contenu structure par page
- Contenu unique par page : descriptions, FAQ et cas d'usage générés par l'IA
- Metadata : titre, description, OG tags uniques par page
- Maillage interne : liens croises entre les pages liees

6.4 **Qualité du contenu a grande echelle**
- Le piege du contenu superficiel : ne généré pas 1000 pages avec 100 mots chacune
- Contenu minimum viable : 500+ mots uniques par page
- Enrichissement par l'IA : longDescription, FAQ, cas d'usage, benefices
- Quality gates : verifier la longueur et l'unicite du contenu avant l'indexation
- noindex pour les pages qui ne passent pas le seuil de qualité

6.5 **Mesurer le succès du pSEO**
- Google Search Console : impressions, clics, pages indexees
- Rapport de couverture : combien de pages indexees vs soumises
- Trafic par template : quels types de pages performent le mieux
- Taux de conversion par type de page

**Exercice** : Créé un système pSEO avec 50+ pages a partir d'un template. Deploie et suis l'indexation.

### MODULE 7 — Link Building & Autorite

**Objectif** : Construire l'autorite de domaine grace a un link building stratégique.

**Lecons a générer :**

7.1 **Pourquoi les liens comptent toujours (et compteront encore longtemps)**
- Liens = votes de confiance d'autres sites
- Google utilise les liens comme facteur de classement top 3
- Qualité > quantité (1 lien d'un site DR70 > 100 liens de sites DR5)

7.2 **Les stratégies de link building qui marchent en 2026**

|-----------|--------|---------|-------------|

| **Recherche originale** | Eleve | Très Elevee | Elevee (une fois publie) |
| **Outil/calculateur** | Eleve | Très Elevee | Très Elevee |
| **Broken link building** | Moyen | Moyenne | Moyenne |
| **Digital PR** | Eleve | Très Elevee | Moyenne |
| **Partenariats de contenu** | Moyen | Elevee | Moyenne |

7.3 **Link building assiste par l'IA**
- Claude pour la redaction d'emails d'outreach
- IA pour trouver des opportunites de liens (liens casses, mentions non-linkees)
- IA pour la personnalisation a grande echelle

7.4 **Stratégie de maillage interne**
