## Philosophie de la Formation

### L'Etat d'Esprit Arbitrage

L'arbitrage en finance = acheter la ou c'est pas cher, vendre la ou c'est cher. L'arbitrage d'outils IA = identifier des outils sous-evalues, les connecter intelligemment, et créer une valeur superieure a la somme des parties.

**Principes :**
1. **Aucun outil n'est parfait seul** — la magie vient des connexions
2. **La vitesse d'adoption = avantage concurrentiel** — celui qui maitrise l'outil en premier gagne
3. **Chaque outil a un "sweet spot"** — l'utiliser pour autre chose = gaspillage
4. **Les integrations font la vraie valeur** — un outil connecte a 5 autres = 10x plus puissant
5. **Toujours tester avant de recommander** — on ne vend pas de la hype, on vend des resultats

### MODULE 2 — Les Modèles IA : Comprendre et Choisir

**Objectif** : L'eleve sait quel modèle utiliser pour quelle tache et comprend les forces/faiblesses de chacun.

**Lecons a générer :**

2.1 **Le paysage des modèles IA en 2026**

Pour chaque modèle, générer une fiche complète :

- **Claude (Anthropic)** : Opus 4.6 (raisonnement profond, code, analyse longue), Sonnet 4.6 (equilibre vitesse/qualité), Haiku 4.5 (rapide, economique)
 - Sweet spot : coding, analyse de documents longs, raisonnement complexe, respect d'instructions
 - API, SDK Python/TypeScript, Claude Code CLI

- **GPT (OpenAI)** : GPT-4.1, o3/o4-mini (reasoning), GPT-4o (multimodal)
 - Sweet spot : génération de texte créatif, API mature, ecosystem large
 - API, plugins, GPTs custom

- **Gemini (Google)** : Gemini 3 Pro/Flash, contexte 1M+ tokens
 - Sweet spot : contexte ultra-long, multimodal (video, audio), integration Google
 - API, AI Studio, Vertex AI

- **Open Source** : Llama (Meta), Mistral, Qwen, DeepSeek
 - Sweet spot : self-hosting, privacy, customisation fine
 - Ollama, vLLM, Together AI

2.2 **Quand utiliser quel modèle — Decision tree**
- Code/dev → Claude (Opus pour complexe, Sonnet pour rapide)
- Texte créatif long → GPT-4.1 ou Claude
- Analyse de video/image → Gemini 3
- Contexte ultra-long (>200K tokens) → Gemini 3 Pro
- Budget serre → Haiku, GPT-4o-mini, Flash
- Privacy/self-hosted → Llama, Mistral via Ollama
- Reasoning/math → o3, Claude Opus

2.3 **Multi-model strategy**
- Pourquoi utiliser plusieurs modèles (arbitrage cout/performance)
- Router pattern : envoyer les taches simples au modèle economique, les complexes au premium
- Fallback chains : si modèle A echoue, essayer modèle B
- Cost optimization : mesurer et optimiser le cout par tache

2.4 **APIs & SDKs — Les bases**
- Comment appeler les APIs : Anthropic, OpenAI, Google
- SDKs : `@anthropic-ai/sdk`, `openai`, `@google/genai`
- Streaming, tool use, structured outputs
- Rate limits et bonnes pratiques

**Exercice pratique** : Tester 3 modèles différents sur la même tache, comparer qualité/vitesse/cout. Documenter.

### MODULE 3 — Les Connecteurs : MCP, APIs, Webhooks

**Objectif** : L'eleve maitrise les 3 modes de connexion entre outils et sait choisir le bon pour chaque situation.

**Lecons a générer :**

3.1 **MCP (Model Context Protocol) — Le standard emergent**
- Rappel : MCP = USB pour l'IA
- Avantages : standardise, decouvrable, composable
- Limitations : encore jeune, pas tous les outils le supportent
- Liste des MCP servers disponibles (mise a jour reguliere)
- Comment verifier si un outil supporte MCP

3.2 **APIs REST/GraphQL — Le classique**
- Comment lire une documentation API en 10 minutes
- Authentication : API keys, OAuth 2.0, Bearer tokens
- Les patterns : GET/POST/PUT/DELETE, pagination, rate limiting
- Outils : Postman/Hoppscotch pour tester, cURL pour scripter
- Quand préférer une API directe vs MCP

3.3 **Webhooks — L'evenementiel**
- Comment ca marche : push vs pull
- Cas d'usage : Stripe paiement → mettre a jour la BDD, GitHub push → deployer
- Sécurité : signature verification, HMAC
- Outils pour tester : ngrok, webhook.site

3.4 **Zapier / Make (Integromat) / n8n — Les no-code connectors**
- Quand utiliser du no-code vs du code
- Avantages : rapide a mettre en place, pas de maintenance serveur
- Inconvenients : cout a l'echelle, limitations, pas assez flexible pour les cas complexes
- n8n self-hosted : la meilleure option pour les CAIOs (gratuit, flexible, self-hosted)

3.5 **Composio — Le meta-connecteur**
- 200+ apps en une integration
- MCP server pour Claude Code
- OAuth gere automatiquement
- Comment l'utiliser comme hub central de connexion

3.6 **Choisir la bonne méthode de connexion**
- Decision tree :
 - L'outil a un MCP server → utiliser MCP
 - Pas de MCP mais bonne API → API directe
 - Besoin de reagir a des evenements → webhooks
 - Prototype rapide / pas de code → Zapier/Make/n8n
 - Multi-outils simultanes → Composio

**Exercice pratique** : Connecter 3 outils différents en utilisant 3 méthodes differentes (MCP, API, webhook).
